Socks5协议

Socks5协议是一个代理协议，旨在为位于Intranet防火墙后的用户提供访问Intranet的代理服务。

关于代理

根据HTTP1.1的定义：

An intermediary program which acts as both a server and a client for the purpose of making requests on behalf of other clients. Requests are serviced internally or by passing them on, with possible translation, to other servers.

代理即中间人，同时满足C/S双方的规范，一人分饰两角，目标服务器眼中的客户端，真正客户端眼中的服务器。当其对两端传输的数据加以修改，则为非透明代理,反之为透明代理。

协议流程

1、认证
认证方法：
  • 0x00: No AUTHENTICATION
  • 0x01: GSSAPI
  • 0x02: USERNAME/PASWWORD
  • 0X03: to X’7F’ IANA ASSIGNED
  • 0x80: to X’FE’ RESERVED FOR PRIVATE METHODS
  • 0XFF: NO ACCEPTABLE METHODS

1.1、客户端 -> 代理服务器，请求认证：

版本号（1字节） 可供选认证方法（1字节） 选择的方法（1-255字节）
固定为5 选了多少种 即上表中的方法

1.2、代理服务器 -> 客户端，响应认证：

版本号(1字节) 确认认证的方法
固定为5 0x00:　无需客户端发送进一步认证信息
0x01:　需要客户端进一步认证
0xFF: 相当于拒绝请求，客户端只能关闭连接
2、请求信息

2.1、客户端 -> 代理服务器，发送目标信息：

版本号(1字节) 命令(1字节) 保留(1字节) 请求类型(1字节) 地址(不定长) 端口(2字节)
固定为５ 0x01: CONNECT
0x02: BIND
0x03:UDP ASSOCIATE		 固定为0x00 0x01: IPV4地址
0x03: 域名
0x04: IPV6地址		 若请求类型是域名，第１个字节为域名长度 目的端口号

其中：

  • CONNECT: 用于客户端请求服务器进行代理
  • BIND: 用于客户端向服务器上报自己的反向连接监听地址（如应用于FTP下载，客户端需要接受来自服务器的连接
  • UDP ASSOCIATE: 用于请求建立到UDP数据报中继的连接

2.2、代理服务器 -> 客户端,确认连接：

版本号(1字节)　 　确认回应(1字节) 保留(1字节) 响应类型（１字节) 地址(不定长) 端口(2字节)
固定为５ 0x00: succeeded
0x01: general SOCKS server failure
0x02:connection not allowed by ruleset
0x03: Network unreachable
0x04: Host unreachable
0x05: Connection refused

0x01: IPV4地址

32.Longest Valid Parentheses

Given a string containing just the characters ‘(‘ and ‘)’, find the length of the longest valid (well-formed) parentheses substring.

Example 1:

Input: “(()”
Output: 2
Explanation: The longest valid parentheses substring is “()”

Example 2:

Input: “)()())”
Output: 4
Explanation: The longest valid parentheses substring is “()()”

题解：
根据题意，可以有两种方法求最长合法括号子串长度:

1.借助栈括号匹配求解

    首先设置变量start,表示合法括号串的起始位置，之后遍历字符串，若遇到左括号，那么将当前下标压入栈中，若遇到右括号，若当前栈为空，则将start更新为下一个坐标，否则栈顶元素出栈，此时栈为空，更新最大值$num = max(num, i - start + 1)$ ，否则$num = max(num, i - m.top())$。

堆排序(heapsort)

发表于 2019-03-04

    堆排序(heapsort)是一种时间复杂度为$O$($nlgn$)的排序算法，其特点是基于一种称为“堆”的数据结构来进行信息管理。该算法具有空间原址性：任何时候都只需要常数个额外元素空间存储临时数据。

一、堆

在学习堆排序算法之前，首先应当了解堆的概念与性质。
如下图，（二叉）堆是一个数组，同时又被看作为一个近似的完全二叉树。每一个节点对应数组中一个元素，除了最底层外，该树完全充满，且从左向右填充。若用数组A来表示，则可设length为数组元素个数，heap-size为堆元素个数。则:
$0<=heap-szie<=length$。

设树的根节点为A[1],则给定一个下标i,容易计算其父节点，左右孩子的下标分别为:

二叉堆可以分为最大堆和最小堆两种形式，在最大堆中，最大堆性质是指除了根以外的所有节点i都应满足:
$A[Parent(i)]\leq [i]$;同理最小堆性质为除了根以外的所有节点都有:$A[Parent(i)]\geq A[i]$。
堆排序算法中使用的为最大堆，最小堆通常用于构造优先队列。

二、算法原理

对于堆排序算法，可以将其分为几个过程分别考虑：

1、MAX-HEAPIFY过程：维护堆的性质

    MAX-HEAPIFY是用于维护最大堆性质的重要过程，其输入为一个数组A，一个下标i以及堆大小heap-size。在调用MAX-HEAPIFY时，假定根节点为LEFT(i)和RIGHT(i)的二叉树都是最大堆，但这时A[i]有可能小于其孩子，此时便违背了最大堆的性质。解决办法是通过让A[i]的值在最大堆中”逐级下降”,从而使得以下标i为根节点的子树重新遵循最大堆的性质。

Manacher’s Algorithm(马拉车算法)

马拉车算法是用来查找字符串最长回文子串的线性方法,该算法十分巧妙地将时间复杂度降低到了$O(n)$

1.算法原理：

首先Manacher算法巧妙地通过添加字符将长度为奇数的回文串与长度为偶数的回文串一起考虑，具体做法为:
在原字符串的每个相邻两个字符中插入一个分隔符，并在首尾也各自增加一个分隔符，
举例如下:

经过转换的字符串T[i]不影响原字符串S[i]中回文串的性质。
Manacher算法关键在于其维护的辅助数组Len[i],其含义是以字符T[i]为中心的最长回文子串的最右字符到T[i]的长度，假设以T[i]为中心的最长回文子串是T[l,r],则$Len[i] = r - i + 1$。
可以证明:$Len[i] - 1$即为以T[i]在原字符串中的长度。
证明如下:
观察字符串T,对于以T[i]为中心的最长回文子串，其长度为$2 * Len[i] -1$,同时可以发现，T中所有的回文子串，其中分隔符#的数量必比字符数量多1，即每个回文子串必有$Len[i]个分隔符和Len[i]
-1$个字符，因此该回文串在原字符串中的长度为$Len[i]-1$。因此根据性质，只要求出Len[i]，即可得到最长回文子串。
从字符串左端出发，依次计算Len[i]，则当计算到Len[i]时，Lenj已知。维护变量P为之前以计算最长回文子串右端点的最大值，同时记录该子串中心位置坐标Po，要求Len[i],则可以考虑以下情况:

3.Longes Substring Without Repeating Characters

Given a string, find the length of the longest substring without repeating characters.

Example 1:

Input: “abcabcbb”
Output: 3
Explanation: The answer is “abc”, with the length of 3.

Example 2:

Input: “bbbbb”
Output: 1
Explanation: The answer is “b”, with the length of 1.

Example 3:

Input: “pwwkew”
Output: 3

Explanation: The answer is “wke”, with the length of 3.
Note that the answer must be a substring, “pwke” is a subsequence and not a substring.
题解：

根据题意，要求根据输入字符串获得没有重复字符的最大连续子串(substring),可以考虑一个滑动窗口，设窗口右端为left,左端为当前位置，同时维护一个大小为128的哈希表，表示每一个键盘输入的字符最近出现的位置。当前指针不断向前移动，比较指针所指字符最近出现位置与left的大小，比left大则更新left，比left小则继续向前。维护一个res,记录并更新每次出现过的窗口大小，最终得到的值即为最长substring的长度。
